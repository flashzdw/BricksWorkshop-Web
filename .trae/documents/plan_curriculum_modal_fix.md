# 课程详情页（VEX 123等）移动端遮挡及Apple风格优化方案

## 1. 当前状态分析
- **遮挡问题**：当前课程详情弹窗的外层容器使用了 `fixed inset-0` 和 `max-h-[90vh]`。在移动端浏览器（如 Safari、微信内置浏览器）中，由于底部导航栏和顶部地址栏的动态变化，`100vh` 往往会超出实际可视区域，导致弹窗底部的内容和关闭按钮被遮挡，用户无法滚动到底部或关闭弹窗。
- **UI风格问题**：
  - 移动端弹窗居中显示，在手机上显得拥挤，不够贴近 iOS 的原生体验。
  - 图片使用了 `object-cover` 且没有内边距，导致产品图被生硬裁剪，背景色与卡片融合度不高。
  - 字体使用了 `font-light` 和 `font-bold` 的极端对比，而 Apple 风格更倾向于使用 `font-normal` 搭配 `font-semibold` 以获得更舒适的阅读体验。
  - 圆角不够大，且关闭按钮的样式略显突兀。

## 2. 拟修改内容 (Proposed Changes)

我们将主要修改 `/workspace/src/components/Curriculum.tsx` 中的弹窗部分。

### 2.1 修复移动端遮挡问题
- 将弹窗外层容器的高度控制改为 `h-[100dvh]`（动态视口高度），以完美适配移动端浏览器的地址栏动态伸缩。
- 在移动端将弹窗布局改为 **Bottom Sheet（底部半屏弹窗）**，使用 `items-end sm:items-center` 让弹窗在手机上吸底显示。
- 在弹窗内部滚动区域底部增加足够的 padding (`pb-10`)，确保内容滚动到底部时不会贴边。

### 2.2 优化 Apple 风格 UI
- **底部抽屉设计**：移动端使用 `rounded-t-[32px]` 配合顶部的小横条（Drag Indicator），打造原生的 iOS Bottom Sheet 体验。桌面端保持居中并使用 `sm:rounded-[32px]`。
- **图片展示优化**：图片容器背景色改为 Apple 常用的高级灰 `#F5F5F7`，并使用 `object-contain` 配合内边距（`p-8`），让机器人产品像在 Apple Store 中一样完整、精致地展示。
- **排版与间距**：
  - 标题改为 `font-semibold tracking-tight`，正文改为 `font-normal`。
  - 统一内容区域的间距（使用 `gap-8`、`space-y-8`），提升呼吸感。
- **背景与控件**：
  - 背景遮罩升级为更强烈的毛玻璃效果 `bg-black/20 backdrop-blur-md`。
  - 关闭按钮（X）添加毛玻璃底色 `bg-white/80 backdrop-blur-md shadow-sm`，使其在任何背景下都清晰可见且优雅。

## 3. 假设与决策
- **假设**：当前项目中已经配置了 Tailwind CSS 并且支持 `dvh` 单位（Tailwind v3.2+ 默认支持）。如果不支持，`100dvh` 会被忽略，但项目中一般都已经支持。
- **决策**：移动端采用 Bottom Sheet（从底部弹出的抽屉）而不是居中弹窗，这是因为 Bottom Sheet 是移动端展示详情的最优解，不仅能解决视口高度遮挡问题，也最符合 Apple 风格的交互规范。

## 4. 验证步骤
1. 修改完成后，在移动端浏览器（Safari / Chrome）中打开页面，点击“VEX 123”课程详情。
2. 验证弹窗是否从底部弹出，并且上下滑动时不会被底部导航栏遮挡。
3. 验证产品图片是否完整展示（未被不合理裁剪），UI 风格是否更加精致、符合 Apple 官网质感。
