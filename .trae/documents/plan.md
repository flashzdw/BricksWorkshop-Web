# 修复网站页面布局与图片加载问题计划

## 1. 探索分析结果
- **“从零基础到世界舞台”部分（Pathway.tsx）**：
  - 现有的连线实现是在全局使用了两根绝对定位的线条（桌面端横线、移动端竖线）。移动端的竖线直接从顶部贯穿到底部，导致其穿过了各项目标题和描述文字，这在视觉上表现为“后两项中间有灰色竖线”（穿模或错误显示）。
  - 圆形光晕使用了 `scale-150` 和 `-z-10`，在特定的层叠上下文中可能被背景遮挡或导致边缘被截断（即“与背景有遮挡”）。
- **首页顶部空白过大（Hero.tsx）**：
  - 手机版首页（Hero组件）本身使用了 `min-h-screen flex items-center` 来实现全屏垂直居中，但在内容容器上又额外叠加了 `mt-16 sm:mt-20`，导致内容整体下移，使得标题上方的留白在移动端显得过大。
- **风采展示图片失效（Gallery.tsx）**：
  - Gallery 组件中引用了 `raw.githubusercontent.com` 的外部图片，这些链接已失效（404）。根据规范，应使用 Trae 内置的 AI 生成图片接口替换这些失效链接。

## 2. 计划更改细节

### 2.1 修改 Pathway.tsx
- **修复连接线**：
  - 移除原有的全局连线 `<div className="md:hidden ... w-0.5">` 和 `<div className="hidden md:block ... h-0.5">`。
  - 在每个 `step` 遍历项内部添加专门的桌面端横线（仅在非最后一个元素上显示），精确定位在图标的垂直居中位置（`top-10`），并将线条延伸到下一个图标的中心，彻底避免穿过文字。
  - 移动端默认隐藏连线，因为堆叠布局不需要容易引起视觉穿模的长竖线。
- **修复光晕遮挡**：
  - 将光晕的实现从 `absolute inset-0 scale-150 -z-10` 更改为更稳定的 `absolute -inset-4 blur-xl`。
  - 在 DOM 结构中将光晕置于图标（具有 `relative z-10`）的前面，避免使用负 `z-index`，确保光晕永远是完整的圆形，并且处于图标和背景之间，不会发生任何遮挡或截断。

### 2.2 修改 Hero.tsx
- **移除多余的顶部外边距**：
  - 将内容容器 `<div className="relative z-10 max-w-7xl mx-auto px-6 sm:px-8 text-center mt-16 sm:mt-20">` 中的 `mt-16 sm:mt-20` 移除，或者替换为极小的补偿间距（如在移动端移除 `mt-16`）。这将使标题整体上移，完美地在屏幕中居中，解决手机版标题上方空白太大的问题。

### 2.3 修改 Gallery.tsx
- **替换失效图片**：
  - 移除原有的 Github 404 图片链接数组。
  - 使用 `https://coresg-normal.trae.ai/api/ide/v1/text_to_image?prompt={prompt}&image_size=landscape_16_9` 生成 5 张符合“课堂风采”主题的教育机器人高清图片。
  - 提示词涵盖：孩子们在课堂组装机器人、使用平板编程、机器人竞赛等场景，以确保完美符合页面主题。

## 3. 验证步骤
1. 打开预览链接，在手机视图下检查首页标题上方的空白是否已经恢复正常，内容是否完美居中。
2. 滚动至“赛事路径”部分，检查：
   - 桌面端下，各圆形图标之间是否有精准的横向连线，且不穿过文字。
   - 移动端下，多余的纵向灰线是否已消失。
   - 所有圆形光晕是否显示完整、自然，没有被背景遮挡或截断的痕迹。
3. 滚动至“课堂瞬间”部分，检查 5 张 AI 生成的高清展示图片是否能正常加载并显示。
